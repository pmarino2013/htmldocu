🚀 Clase 4: Formularios y Tablas en HTML

Tema: Recolección de datos y organización de información

🎯 Objetivo de la clase

Al finalizar la clase, el estudiante podrá:

  • Crear formularios funcionales en HTML
  • Entender los principales tipos de inputs
  • Usar validaciones nativas
  • Crear tablas correctamente estructuradas
  • Incorporar estos elementos en su Biografía Profesional

🧠 1. ¿Qué es un formulario?

Un formulario es la forma que tiene una página web de recibir información del usuario.

Ejemplos reales:

  • Formulario de contacto
  • Registro en una plataforma
  • Inicio de sesión
  • Suscripción a newsletter

🧱 2. Estructura básica de un formulario

La etiqueta principal es:

<form>
</form>

Pero sola no hace nada. Necesita inputs (campos de entrada).

🔹 Ejemplo mínimo

<form>
  <input type="text">
</form>

Eso crea un campo donde el usuario puede escribir.

📌 3. Los tipos de Input más importantes

📝 1️⃣ Texto

<input type="text">

Sirve para nombres, ciudades, etc.

📧 2️⃣ Email

<input type="email">

Valida automáticamente que tenga formato de correo.

Si el usuario escribe:

juan@gmail

El navegador mostrará error.

🔢 3️⃣ Número

<input type="number">

Solo permite números.

🔒 4️⃣ Password

<input type="password">

Oculta lo que el usuario escribe.

📅 5️⃣ Fecha

<input type="date">

Muestra selector de calendario.

📋 6️⃣ Área de texto (mensajes largos)

<textarea> </textarea>

Ideal para mensajes de contacto.

🏷️ 4. La etiqueta <label> (Muy importante)

Siempre debemos vincular el texto con el input.

❌ Incorrecto:

Nombre
<input type="text">

✅ Correcto:

<label for="nombre">Nombre</label>
<input type="text" id="nombre">

Esto mejora:

  • Accesibilidad
  • Experiencia de usuario
  • Profesionalismo

🛑 5. Validaciones Nativas

HTML permite validar sin usar JavaScript.

🔹 Campo obligatorio

<input type="text" required>

🔹 Longitud mínima

<input type="password" minlength="6">

🔹 Patrón personalizado

<input type="text" pattern="[0-9]{10}">

Ejemplo: teléfono de 10 números.

📬 6. Botón de envío

<button type="submit">Enviar</button>

O también:

<input type="submit" value="Enviar">

🧩 7. Ejemplo completo de formulario de contacto

<section>
  <h2>Contacto</h2>

  <form>

    <label for="nombre">Nombre</label>
    <input type="text" id="nombre" required>

    <br> <br>

    <label for="email">Email</label>
    <input type="email" id="email" required>

    <br> <br>

    <label for="mensaje">Mensaje</label>
    <textarea id="mensaje" required> </textarea>

    <br> <br>

    <button type="submit">Enviar</button>

  </form>
</section>

🎯 ¿Qué es la propiedad pattern?

pattern es un atributo de los <input> que permite definir una regla personalizada de validación usando expresiones regulares (RegEx).

En palabras simples:

Es una forma de decirle al navegador:
“Este campo solo acepta texto que cumpla esta forma específica”.

🧠 ¿Para qué sirve?

Sirve para validar formatos específicos cuando:

  • El tipo email no es suficiente
  • Necesitamos un formato exacto
  • Queremos controlar cómo se escribe algo

🧱 Sintaxis básica

<input type="text" pattern="REGEX_AQUI">

Ejemplo simple:

<input type="text" pattern="[0-9]{10}">

Esto significa:

  • Solo números
  • Exactamente 10
  • Ni más, ni menos

🔎 ¿Cómo funciona?

pattern usa expresiones regulares (RegEx).

No vamos a profundizar demasiado en RegEx ahora (eso sería otra clase 😉), pero vamos a entender lo básico.

📘 Ejemplos prácticos y explicados

📞 1️⃣ Teléfono de 10 números

<input type="text" pattern="[0-9]{10}" required>

¿Qué significa?

[0-9] → solo números

{10} → exactamente 10 veces

✔ Válido:

3814567890

❌ Inválido:

381-456-7890

🔠 2️⃣ Solo letras

<input type="text" pattern="[A-Za-z]+" required>

Significa:

  • Solo letras
  • Al menos una

✔ Válido:

Juan

❌ Inválido:

Juan123

🆔 3️⃣ DNI argentino (7 u 8 números)

<input type="text" pattern="[0-9]{7,8}" required>

Significa:

  • Solo números
  • Entre 7 y 8 caracteres

🔐 4️⃣ Contraseña con mínimo 8 caracteres, letras y números

<input 
  type="password" 
  pattern="(?=.*[A-Za-z])(?=.*[0-9]).{8,}" 
  required>

Este es más avanzado, pero significa:

  • Al menos una letra
  • Al menos un número
  • Mínimo 8 caracteres

🧠 Algo MUY importante

pattern:

✔ Solo funciona cuando el formulario se envía
✔ No reemplaza required
✔ No muestra mensaje personalizado (por defecto)

Si el valor no cumple el patrón, el navegador mostrará:

“El formato no es válido”

🎨 Cómo mejorar la experiencia (tip profesional)

Podemos agregar el atributo title:

<input 
  type="text" 
  pattern="[0-9]{10}" 
  title="Debe contener exactamente 10 números"
  required>

Ahora el navegador mostrará ese mensaje cuando el formato no sea correcto.

💡 Esto es MUY buena práctica.

🧠 ¿Cuándo usar pattern?

Usarlo cuando:

  • Necesitamos un formato exacto
  • El input type no cubre nuestra necesidad
  • Queremos controlar estructura específica

Ejemplos reales:

✔ Código postal
✔ Número de cliente
✔ DNI
✔ Teléfono sin guiones
✔ Usuario sin espacios

🚫 ¿Cuándo NO usarlo?

No usarlo cuando:

  • El tipo ya valida solo (ej: type="email")
  • La validación es demasiado compleja
  • Necesitamos mensajes personalizados avanzados (ahí usamos JavaScript)

📌 Comparación rápida

Atributo Qué valida
type="email" Formato básico de email
required Que el campo no esté vacío
minlength Cantidad mínima de caracteres
pattern Formato personalizado exacto

🧑‍🏫 Cómo explicarlo

“El pattern es como poner una plantilla encima del texto.
Si lo que escribís no encaja en la plantilla… no pasa.”

🎯 Resumen

pattern sirve para:

  • Crear reglas personalizadas
  • Validar formatos específicos
  • Profesionalizar formularios
  • Evitar datos mal escritos

Es una herramienta simple… pero poderosa.

📊 8. Tablas en HTML

Las tablas sirven para mostrar datos estructurados.

Ejemplo:

  • Estudios realizados
  • Experiencia laboral
  • Horarios
  • Listado de precios

🧱 Estructura básica de una tabla

<table>
</table>

Pero necesita filas y columnas.

🔹 Filas

<tr>
</tr>

🔹 Celdas normales

<td>
</td>

🔹 Celdas de encabezado

<th>
</th>

🧩 Ejemplo completo

<table border="1">

  <tr>
    <th>Año</th>
    <th>Institución</th>
    <th>Título</th>
  </tr>

  <tr>
    <td>2023</td>
    <td>Academia IT</td>
    <td>Desarrollo Web</td>
  </tr>

  <tr>
    <td>2021</td>
    <td>Universidad X</td>
    <td>Analista</td>
  </tr>

</table>

📌 Explicación simple

  • <table> → la tabla
  • <tr> → una fila
  • <th> → encabezado
  • <td> → datos

⚠️ Errores comunes de principiantes

❌ No cerrar <tr>

❌ Mezclar <th> y <td> incorrectamente

❌ Usar tablas para hacer layout (eso ya no se hace)

🧠 Concepto profesional importante

Las tablas se usan para:

✔ Datos
❌ No para diseño

El diseño se hace con CSS (lo veremos en próximas semanas).
