HTML (HyperText Markup Language)

1. Creación del archivo y estructura básica

Para empezar, solo necesitas un editor de código (como VS Code).

  1. Crea una carpeta para tu proyecto.
  2. Dentro, crea un archivo llamado index.html.
  • Tip: Usamos index porque los servidores buscan este nombre por defecto para mostrar la página principal.

La estructura mínima obligatoria que verás en cualquier sitio profesional es esta: 

<!DOCTYPE html>
<html lang="es">
<head>
    </head>
<body>
    </body>
</html>

2. La etiqueta y los Metadatos en 2026

El <head> es el "cerebro" oculto. Aquí es donde le dices a los navegadores y buscadores de qué trata tu página. En 2026, con el auge de la IA y el SEO semántico, estos son los elementos imprescindibles:

  • charset="UTF-8": Fundamental para que se lean bien las tildes y la "ñ".
  • viewport: Crucial para que tu web se vea bien en móviles. Sin esto, el sitio se verá diminuto en un iPhone o Android.
  • title: Es el nombre que aparece en la pestaña del navegador.
  • description: Sigue siendo vital para que Google (y los agentes de IA) resuman tu contenido.

Ejemplo de un moderno y optimizado:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Primera Web Semántica</title>
    <meta name="description" content="Aprende HTML desde cero con las mejores prácticas de 2026.">

    <link rel="stylesheet" href="styles.css">
</head>

3. El atributo lang: ¿Por qué importa tanto?

Seguro notaste el <html lang="es">. No es solo un detalle estético.

  • Accesibilidad: Los lectores de pantalla para personas con discapacidad visual lo usan para saber qué acento y pronunciación aplicar.
  • SEO: Ayuda a los motores de búsqueda a segmentar tu audiencia por idioma.
  • Traducción automática: Permite que navegadores como Chrome ofrezcan traducir la página si el usuario habla otro idioma.

4. Semántica: El corazón del HTML moderno

Antiguamente, todo se hacía con etiquetas <div>. Hoy, eso es una mala práctica. La semántica significa usar etiquetas que tengan significado tanto para los humanos como para las máquinas.

Etiqueta Propósito
Introducción de la página o sección (logos, navegación).
Contenedor de enlaces de navegación principal.
El contenido único y principal de esa página (solo debe haber uno).
Contenido independiente (como un post de blog o una noticia).
Agrupa contenido relacionado temáticamente.
El pie de página con información de contacto o enlaces legales.

5. Tu primer cuerpo de página ()

Aquí es donde ocurre la magia visible. Un ejemplo semántico sencillo se vería así: 

<body>
    <header>
        <h1>Mi Blog de Tecnología</h1>
        <nav>
            <ul>
                <li> <a href="#">Inicio</a> </li>
                <li> <a href="#">Cursos</a> </li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <h2>Aprendiendo HTML en 2026</h2>
            <p>La semántica es la clave para que la IA entienda mi contenido.</p>
        </article>
    </main>

    <footer>
        <p> &copy; 2026 - Creado con pasión por el código.</p>
    </footer>
</body>

Resumen para recordar:

  1. UTF-8 y Viewport son los metadatos que no pueden faltar.
  2. lang="es" es un estándar de oro para la accesibilidad.
  3. Usa etiquetas semánticas en lugar de abusar del <div>. Tu SEO te lo agradecerá.

Emmet

Es un motor de abreviaciones integrado en VS Code que te permite escribir líneas enteras de código usando una sintaxis corta. Escribes un "atajo", presionas Tab (o Enter), y Emmet lo expande mágicamente en código HTML completo.

1. El truco de magia inicial: El signo de exclamación

  • Escribe: !
  • Presiona: Tab
  • Resultado: VS Code genera toda la estructura inicial de un documento HTML5 con los metadatos necesarios para 2026.

2. Los atajos más útiles (Sintaxis Emmet)

Emmet funciona con símbolos lógicos. Aquí tienes los más comunes para el día a día:

A. Creación de etiquetas simples

Solo escribe el nombre de la etiqueta sin los signos < >.

  • h1 + Tab ➜ <h1></h1>
  • p + Tab ➜ <p></p>

B. Hijos y hermanos (Estructura)

  • Hijo (>): Para meter una etiqueta dentro de otra.

  • nav>ul>li ➜ Crea un menú con una lista y un ítem dentro.

  • Hermano (+): Para poner etiquetas al mismo nivel.

  • h1+p ➜ Crea un título y un párrafo debajo.

C. Multiplicación (*)

¿Necesitas una lista con 5 elementos? No los copies y pegues.

  • ul>li*5 ➜ Crea una lista <ul> con cinco <li> adentro automáticamente.

D. Clases e IDs (Sintaxis CSS)

  • Clase (.):section.contenido ➜ <section class="contenido"></section>
  • ID (#):div#principal ➜ <div id="principal"></div>
  • Combinado:footer.oscuro#pie-web ➜ <footer class="oscuro" id="pie-web"></footer>

3. Un ejemplo "Pro"

Imagina que quieres crear una sección de noticias con un título, una imagen y un párrafo. En lugar de escribir 10 líneas, escribes esto:

section.noticias>h2+img+p

Y al presionar Tab, Emmet hace esto por ti:

<section class="noticias">
    <h2> </h2>
    <img src="" alt="">
    <p> </p>
</section>

4. El "Lorem Ipsum" instantáneo

A veces necesitas texto de relleno para ver cómo queda el diseño.

  • Escribe lorem + Tab y generará un párrafo de texto en latín.
  • Escribe lorem10 + Tab y generará exactamente 10 palabras.

¿Por qué deberías usarlo siempre?

  1. Velocidad: Escribes un 70% menos de caracteres.
  2. Precisión: Evitas errores comunes como olvidar cerrar una etiqueta (Emmet las cierra siempre por ti).
  3. Estándar: Es el lenguaje universal entre desarrolladores senior.
